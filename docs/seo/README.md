# SEO / AI-SEO knowledge base

These two documents are the source-of-truth for ChatAdmin's SEO feature:

- **`SKILL.md`** — the condensed, action-oriented skill (workflow + guardrails).
- **`WordPress-SEO-and-AI-SEO-Master-Guide.md`** — the full playbook.

## How it maps into the plugin

ChatAdmin has no "skills" runtime (it's tool calls + a system prompt), so the
guidance is split three ways:

| Guide concept | Where it lives in code |
|---|---|
| Audit ("15-minute AEO audit", indexing/permalinks/robots/llms.txt checks) | `ChatAdmin\Seo::audit()` → exposed as the read-only `seo_audit` tool (`includes/class-tools.php`) |
| Fixable settings (indexing, permalinks, AI-crawler access, llms.txt, title/tagline) | `seo_setting` kind on `ChatAdmin\SeoBackend`, via `preview_content_change`/`apply_content_change` |
| Per-post title / meta description | `seo_meta` kind (Yoast / Rank Math / SEOPress; AIOSEO → admin handoff) |
| robots.txt AI-crawler `Allow` rules | `Seo::filter_robots_txt()` (`robots_txt` filter), gated by the `chatadmin_seo_allow_ai_crawlers` option |
| `/llms.txt` | `Seo::maybe_serve_llms_txt()` virtual route, content in the `chatadmin_llms_txt` option; generated by `Seo::generate_llms_txt()` |
| Assistant behavior (audit first, fix vs. advise, thresholds) | `# SEO & AI-SEO (AEO/GEO)` section in `Rest::system_prompt()` |

Everything else in the guide (hosting, Core Web Vitals, schema beyond the SEO
plugin, keyword research, GSC/Bing submission, backlinks, GA4) is **advisory** —
WordPress can't change it, so the assistant relays the recommendation and hands
the user a deep link instead of claiming a fix.

When updating these docs, keep the prompt section and `Seo::audit()` checks in sync.
