the objection, answered

"But won't they break something?"

Correct fear. It's exactly what the product is designed around. This isn't admin access with the safety off — it's guardrailed access.

  • Preview → confirm on every change.

    Nothing mutates silently. They see it, then approve it.

  • One item at a time. No bulk-destructive anything.

    There's no "delete all orders" button for a panicked client to find — it's an enforced constraint, not a setting.

  • It hands off instead of improvising.

    Out of scope? It returns a direct link to the right admin page — it never invents a destructive workaround.

"It's not access. It's guardrailed access — the difference between handing a client the keys and handing them a loaded gun."

The whole design goal: let the non-expert act, without being able to blow up the site.

install

Live on a client site in four steps.

  1. Download the latest wpchat-vX.Y.Z.zip from GitHub Releases.
  2. Upload & activate in WP admin → Plugins → Add New → Upload Plugin.
  3. Add an Anthropic API key from console.anthropic.com in the first-run setup.
  4. Send your client to /wpchat — and stop answering the "can you just…" emails.
